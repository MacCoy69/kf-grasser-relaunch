# Grasser KFZ – Website Relaunch: Planung & Konzept

## Context

Die bestehende Website kfz-grasser.de läuft auf Jimdo (CMS-Builder, stark limitiert) und entspricht weder technisch noch visuell dem Anspruch des Unternehmens. Grasser Cars · Sales · Services ist ein Spezialwerkstatt- und Händlerbetrieb mit echter Expertise in Porsche, Audi RS, Sport- und Premiumfahrzeugen – das kommt auf der aktuellen Website kaum rüber. Der Relaunch soll eine moderne, performante Website schaffen, die solvente Käufer von Sportwagen und Luxusfahrzeugen anspricht. Referenz-Design: [stimpfig.de](https://stimpfig.de/).

## Phase 2 – Design-Konzept (abgeleitet aus stimpfig.de-Analyse)

### Zielgruppe

**Primär:** Solvente Käufer von Sportwagen und Premiumautos (Porsche, Audi RS, Mercedes AMG, BMW M). Typischerweise 35–60 Jahre, unternehmerisch oder leitend tätig, kaufen Autos als Passion/Lifestyle.

**Sekundär:** Bestandskunden, die hochwertige Werkstattleistung für ihre Sportwagen suchen (nicht Billigwerkstatt).

### Design-Richtung (inspiriert von stimpfig.de)

| Element | Ansatz |
|---------|--------|
| Farbpalette | Weiß/Hellgrau Hintergrund, Dunkelgrau/Schwarz Typographie, dezente Akzentfarbe (z.B. Anthrazit oder Gold) |
| Fotografie | Hochwertige Fahrzeugfotos, Werkstatt-Atmosphäre, Fabian Grasser persönlich |
| Typographie | Serifenlose, klare Schriften (z.B. Inter, Neue Haas Grotesk oder ähnlich) |
| Sprache | Emotionslos-elegant, Käufer-Intelligenz voraussetzen, kein Urgency-Sprech |
| Navigation | Flat (6 Punkte max), sticky, kein Dropdown-Chaos |
| Hero | Card-basiert mit Service-Pillars (nicht Bild-Slider) ODER großes Fahrzeugfoto mit minimalem Text |
| Inventory | Grid-Layout, 3–4 Spalten Desktop, Bildkarten ohne Preis sichtbar (Kontakt zum Preis) |
| Zweisprachig | DE/EN Toggle |

### Was NICHT gebaut werden soll (Luxus-Anti-Pattern)

- Kein "JETZT ANGEBOT SICHERN!" / Countdown-Timer
- Keine Popup-Formulare
- Kein billiger Testimonial-Block mit Stockfotos
- Keine Finanzierungsrechner (signalisiert falsches Kundensegment)
- Kein Autowechsel-Slider (veraltet)
- Keine grellen Farben / Sale-Badges

## Phase 5 – Tech Stack (Entscheidung ausstehend)

Tech-Stack-Entscheidung wird in separatem Schritt getroffen. Empfehlung vorbereitet:

**Option A – Next.js (App Router) + Headless CMS (z.B. Sanity oder Contentful)**
- Maximale Performance (Static Generation)
- Vollständige Designfreiheit
- Inventory als CMS-Collection verwaltbar
- Deployment: Vercel
- Vorteil: Kunde kann Fahrzeuge selbst verwalten
- Nachteil: Entwicklungsaufwand höher

**Option B – Astro + Decap CMS (ehemals Netlify CMS)**
- Noch performanter (0 JS by default)
- Einfacheres CMS für kleinen Betrieb
- Git-basiertes Content Management
- Deployment: Netlify oder Vercel

**Option C – WordPress + Custom Theme (Blocksy/Genesis)**
- Vertrauteres Backend für Kunden
- Mehr Plugins verfügbar
- Performance-Kompromisse
